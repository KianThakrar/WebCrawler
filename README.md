# WebCrawler – COMP3011 Search Engine

A command-line search engine that crawls [quotes.toscrape.com](https://quotes.toscrape.com/), builds an inverted index with TF-IDF and BM25 scoring, and provides ranked full-text search with phrase matching and query suggestions.

## Algorithms & Data Structures

### Inverted Index

The core data structure maps each unique term to a **posting list** — a dict of URLs, each with three fields:

```json
{
  "wisdom": {
    "https://quotes.toscrape.com/page/1/": {
      "frequency": 3,
      "positions": [12, 45, 89],
      "tf_idf": 0.4812
    }
  }
}
```

| Field | Purpose |
|---|---|
| `frequency` | Raw term count — used by TF-IDF and BM25 |
| `positions` | Token offsets — enables phrase and proximity search |
| `tf_idf` | Pre-computed score — O(1) retrieval at query time |

### TF-IDF Scoring

```
TF(t, d)  = 1 + log(freq(t, d))        # log-normalised to dampen outliers
IDF(t)    = log((N + 1) / df(t))       # smoothed to avoid division by zero
TF-IDF    = TF × IDF
```

Pre-computed at index build time so query-time retrieval is O(1) per posting.

### BM25 Ranking (Okapi BM25)

The industry-standard probabilistic ranking function used by Elasticsearch and Apache Lucene:

```
IDF(t)    = log((N − df + 0.5) / (df + 0.5) + 1)
TF_n(t,d) = freq × (k1 + 1) / (freq + k1 × (1 − b + b × dl/avgdl))
score     = Σ IDF(t) × TF_n(t, d)
```

Default parameters: `k1 = 1.5`, `b = 0.75`. BM25 improves on TF-IDF by:
- **Frequency saturation** (`k1`): additional occurrences contribute diminishing returns
- **Length normalisation** (`b`): penalises long documents that match by chance

### Phrase Search

Uses the stored `positions` lists to verify that query tokens appear **consecutively** in the document. For phrase `"good friends"`, the engine checks that `positions["good"]` contains some value `p` and `positions["friends"]` contains `p + 1`. This is exact phrase matching, not proximity — order matters.

**Complexity:** O(k × D × P) where k = phrase length, D = documents containing first term, P = positions of first term.

### Proximity Scoring

Extends AND search with a position-aware bonus:

```
score = Σ TF-IDF(term, doc) + 1 / (1 + min_distance)
```

`min_distance` is the minimum token offset gap between any two consecutive query terms. This rewards pages where query terms co-occur locally rather than being scattered.

### Query Suggestions (Did-You-Mean)

Uses **Levenshtein edit distance** with a full DP matrix (O(|a| × |b|)) to find vocabulary words within 2 edits of the query. Results are sorted by edit distance then alphabetically. Activates automatically when `find` returns no results.

### Complexity Summary

| Operation | Time | Space |
|---|---|---|
| `build_index` (P pages, T tokens/page) | O(P × T) | O(V × D) |
| `compute_tfidf` (V terms, D docs/term) | O(V × D) | O(1) in-place |
| `find` / `find_bm25` (k terms) | O(k × D + D log D) | O(D) |
| `find_phrase` (k terms, P positions) | O(k × D × P) | O(1) |
| `find_with_proximity` | O(k × D × P²) | O(D) |
| `suggest_terms` (V vocab, W word len) | O(V × W²) | O(W²) |
| `save_index` / `load_index` | O(V × D) | O(V × D) |

### Web Crawler

BFS over pagination links (`li.next > a`). A **deque** provides O(1) queue operations. A **set** tracks visited URLs for O(1) deduplication. A mandatory `time.sleep(6)` between requests enforces the politeness window required by the coursework specification.

## Usage

Start the interactive shell:

```bash
python -m src.main
```

### Commands

| Command | Description |
|---|---|
| `build` | Crawl the website, build the index, save to `data/index.json` |
| `load` | Load a previously built index from `data/index.json` |
| `print <word>` | Print the full inverted index entry for a word |
| `find <word(s)>` | Find pages containing **all** given words, ranked by TF-IDF |
| `quit` / `exit` | Exit the shell |

### Example Session


## Advanced Commands

These features are available programmatically via `SearchEngine`:

```python
from src.indexer import load_index
from src.search import SearchEngine, suggest_terms

idx = load_index("data/index.json")
engine = SearchEngine(idx)

# Exact phrase matching (positional)
engine.find_phrase("good friends")

# Proximity-boosted AND search
engine.find_with_proximity(["good", "friends"])

# BM25 ranking (Okapi BM25)
engine.find_bm25(["wisdom", "courage"])

# Did-you-mean suggestions
suggest_terms(idx, "frends", max_results=3)
```
